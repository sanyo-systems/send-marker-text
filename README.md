# 記録計通信ソフト Python版

## 1. 概要

本システムは、現場で出力されるCSVファイルを監視し、必要な情報を抽出して記録計へ送信するための社内用ツールです。  
既存のVB版記録計通信ソフトの運用をもとに、Pythonで再構築したものです。

主な目的は以下の通りです。

- CSVファイルの変更を自動検知する
- 指示書Noや処理情報を取得し、対象記録計へ送信する
- 送信成功時・失敗時のファイル整理を自動化する
- 重複送信を防止する
- 障害時の切り分けをしやすくする

---

## 2. 主な機能

本システムには、以下の機能があります。

### 2-1. CSV監視
指定フォルダ内のCSVファイルを監視し、更新があったファイルを検知します。

### 2-2. 記録計送信
CSVから必要な値を読み取り、対象の記録計へ送信します。

### 2-3. 送信結果に応じたファイル移動
送信結果に応じて、CSVファイルを以下のフォルダへ移動します。

- 送信成功時: `DONE`
- 送信失敗時: `ERROR`

### 2-4. 再試行処理
送信失敗時は一定回数リトライを行います。

### 2-5. 重複送信防止
送信済みデータの履歴を保持し、同じデータを繰り返し送信しないようにしています。

### 2-6. ログ出力
送信開始、送信成功、送信失敗、CSV移動、再試行などの処理状況をログに出力します。

---

## 3. 想定している運用

本システムは、以下のような運用を想定しています。
---
1. 現場または関連システムがCSVファイルを出力する  
2. 本システムがCSVの変更を検知する  
3. CSVから指示書No、開始時刻、処理情報などを取得する  
4. 対象記録計へ送信する  
5. 送信成功時は `DONE` へ移動する  
6. 送信失敗時は `ERROR` へ移動する  
7. 必要に応じて履歴やログで追跡する  
※　監視するCSVやデータを入れるaccessはsetting.iniで設定する
※　このsetting.iniはmain.pyやmain.exe等の動作させるファイルやソフトと同じ場所に入れる。
---

## 4. フォルダ構成
```text
send-marker-text/
├─ communication/    # 通信処理関連
├─ csv_monitor/      # CSV監視・送信制御
├─ database/         # Access連携処理
├─ history/          # 履歴管理
├─ monitoring/       # ログ・監視補助
├─ ui/               # UI関連
├─ utils/            # 共通処理
├─ main.py           # メイン処理
├─ main.spec         # exe化設定
└─ dummy_recorder.py # テスト用ダミー受信機
```
## 5. 主なファイル・役割
main.py
システム全体の起動処理。

csv_monitor/csv_watcher.py
CSV変更検知、CSV読込、送信キュー投入、送信処理制御の中心です。

communication/send_queue.py
送信キューの管理とワーカー処理を担当します。

communication/recorder_client.py
記録計への通信処理を担当します。

database/access_writer.py
必要に応じてAccess DBへの書き込みを行います。

utils/csv_utils.py
CSVの DONE / ERROR への移動処理などを行います。

dummy_recorder.py
テスト時に記録計の代わりとして動かす確認用プログラムです。

## 6. 設定ファイルについて
本システムは設定ファイルを用いて動作します。
主に以下のような情報を設定します。

CSV監視フォルダ
Access DB の場所
記録計IPアドレス
記録計ポート
対象CSVファイル名
その他運用用設定

注意
実運用用の設定ファイルであるsetting.iniには、以下のような情報が含まれます。

実機IPアドレス
共有DBのパス
実運用用フォルダ情報

そのため、実運用用設定ファイルはGitHubに含めず、配布時は別管理とする運用を推奨します。

## 8. 導入時の考え方
導入時は、以下のような形を想定しています。

プログラム本体は対象PCへ配置する
共有で使用するDBは共有ファイル上のものを利用する
設定ファイルで、対象PCごとの監視先や送信先を調整する
必要に応じて試験用設定と本番用設定を分ける

## 9. 導入時の確認項目

導入前には、少なくとも以下を確認してください。

### 9-1. フォルダ・権限
CSV監視対象フォルダが存在するか
対象フォルダに読み書き可能か
DONE / ERROR フォルダを作成・利用できるか

### 9-2. 設定
記録計IPアドレス・ポートが正しいか
対象CSVファイル名が正しいか
Access DB パスが正しいか

### 9-3. ネットワーク
対象PCから記録計へ疎通できるか
共有DBへアクセスできるか

### 9-4. セキュリティ
セキュリティソフト等により exe 実行が妨げられないか

## 10. 試験時の確認ポイント

試験時には、以下を確認してください。

### 正常系
CSV変更を検知できる
送信キューへ投入できる
記録計へ送信できる
DONE へ移動できる
履歴が保存される

### 異常系
失敗時に再試行する
最終失敗時に ERROR へ移動する
失敗情報が残る

### 重複防止

同じデータを再度送ろうとしたときに再送されない
